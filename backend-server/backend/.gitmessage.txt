# [사용 가능한 type 목록]
# - init     : 초반 환경설정
# - feat     : 새로운 기능 추가
# - fix      : 버그 수정
# - refactor : 코드 리팩토링 (기능 변화 없음)
# - docs     : 문서 수정 (README, 주석 등)
# - design   : CSS 등 UI 디자인 변경
# - comment  : 주석 추가 및 수정
# - rename   : 파일/폴더명 수정, 이동
# - remove   : 파일 삭제
# - deploy   : 배포 관련 작업

# [제목 및 본문 작성 규칙]
# - type은 영어 소문자로 작성
# - 제목은 한글 기준 20자 이내로 간결하게 작성
# - 마침표 및 특수문자 금지 (예: !, .)
# - 제목(type: 제목)과 본문 사이 한 줄 띄우기
# - 본문은 변경한 내용과 이유를 리스트로 작성 (무엇을 왜 했는지 위주)
